# Support

Thank you for using `dbt-checks`.

This document explains where to ask questions, report problems, and request features.

---

# Questions & Discussions

For:
- usage questions
- implementation ideas
- architecture discussions
- grouped validation questions
- adapter compatibility discussions

please use:
- GitHub Discussions (if enabled)
- GitHub Issues

---

# Bug Reports

If you encounter a bug, please open a GitHub issue including:

- dbt version
- adapter version
- package version
- reproduction steps
- expected behavior
- actual behavior
- relevant logs or screenshots (if applicable)

Clear reproduction steps help resolve issues faster.

---

# Feature Requests

Feature requests are welcome.

Please include:
- use case
- expected behavior
- example configuration
- business context if relevant

This helps evaluate how the feature fits the package philosophy and architecture.

---

# Security Issues

Please do **not** report security vulnerabilities through public GitHub issues.

See `SECURITY.md` for vulnerability reporting guidance.

---

# Supported Scope

`dbt-checks` primarily focuses on:

- reusable data quality checks
- grouped validations
- business-oriented semantics
- standardized outputs
- dbt compatibility
- cross-warehouse support through dbt dispatch

Support for highly adapter-specific behavior may be best-effort.

---

# Best-effort Maintenance

`dbt-checks` is maintained on a best-effort basis.

Response times may vary depending on:
- issue complexity
- maintainer availability
- project priorities

Community contributions are welcome and appreciated.
